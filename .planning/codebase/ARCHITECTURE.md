# Architecture

**Analysis Date:** 2026-03-25

## Overview

A static portfolio/resume site built with Astro + Svelte, rendered entirely at build time (SSG) and deployed to GitHub Pages. All content is embedded as TypeScript data; there is no backend, database, or API.

## Architecture Style

- **Pattern:** Static Site Generation (SSG) — Multi-Page Application (MPA)
- **Data flow:** Build-time only. Content lives in `src/i18n/translations.ts` as a typed `const` object → pages import the translation helper → Astro renders HTML at `astro build` → `dist/` is deployed as a fully static artifact.
- **Interactivity:** Minimal. The only interactive island is `LanguageSwitcher.svelte`, hydrated as a Svelte component within the otherwise static Astro shell.

## Key Components

| Component | Role | Location |
|-----------|------|----------|
| `BaseLayout.astro` | HTML shell, `<head>` meta, Open Graph, JSON-LD structured data, font preloads, canonical/hreflang links | `src/layouts/BaseLayout.astro` |
| `index.astro` (pt-BR) | Default locale page — full resume rendered in Portuguese | `src/pages/index.astro` |
| `en/index.astro` | English locale page — same resume structure, English strings | `src/pages/en/index.astro` |
| `LanguageSwitcher.svelte` | Interactive locale toggle link (globe icon + label); receives `currentLang` and `alternatePath` as props | `src/components/LanguageSwitcher.svelte` |
| `translations.ts` | All user-facing copy in both locales (`pt-br` and `en`) typed with `as const`; exports `Locale` type | `src/i18n/translations.ts` |
| `utils.ts` | i18n helpers: `useTranslations`, `getLangFromUrl`, `getAlternateUrl`, `getHtmlLang` | `src/i18n/utils.ts` |
| `global.css` | Single global stylesheet — CSS custom properties, typography, layout, responsive, print | `src/styles/global.css` |

## Page / Route Structure

```
/portfolio/           →  src/pages/index.astro       (default locale: pt-br)
/portfolio/en/        →  src/pages/en/index.astro     (English locale)
```

- Astro's built-in i18n routing is configured with `prefixDefaultLocale: false`, so the Portuguese version has no locale prefix.
- Both pages share identical markup structure; only the language constant and translated strings differ.
- Sitemap is auto-generated by `@astrojs/sitemap` at build time.

## Data Layer

- **Data sources:** Static TypeScript object in `src/i18n/translations.ts`. No external APIs, CMS, or database.
- **Data fetching strategy:** None — all data is imported directly as ES module constants at build time.
- **Content shape:** Nested `as const` object keyed by locale (`pt-br` | `en`), with sub-keys: `meta`, `nav`, `header`, `summary`, `skills`, `experience`, `education`, `footer`, `a11y`.
- **Caching:** Not applicable (fully static output).

## Rendering Strategy

- **Full SSG:** Every page is pre-rendered to static HTML by `astro build`. No server-side rendering occurs at request time.
- **Svelte island:** `LanguageSwitcher.svelte` uses Astro's component island model. Svelte 5 runes (`$props()`, `$derived()`) are used for reactivity; the component renders an `<a>` tag so it functions without JavaScript.
- **Stylesheet inlining:** `build.inlineStylesheets: 'always'` in `astro.config.mjs` causes all CSS to be inlined into the `<style>` block of each HTML file — no separate CSS requests.
- **Fonts:** Self-hosted WOFF2 subsets (`dm-sans-latin.woff2`, `source-serif-4-latin.woff2`) defined in `@font-face` rules in `global.css` and preloaded via `<link rel="preload">` in `BaseLayout.astro`.

## Notable Architectural Decisions

- **No content duplication via shared layout:** Both locale pages are fully self-contained `.astro` files that call the same `BaseLayout`. They share no page-level Astro slot fragments — content sections are written inline in each page file. This means any structural change to the page template must be applied to both `src/pages/index.astro` and `src/pages/en/index.astro`.
- **Locale routing is filesystem-based:** Portuguese at root (`/`), English under `/en/`. Astro's i18n config enforces this without any dynamic routing.
- **Single source of truth for copy:** All translatable strings live in one file (`src/i18n/translations.ts`). Adding a new locale requires adding a new key there and a new page directory under `src/pages/`.
- **JSON-LD structured data** (`Person` schema) is emitted inline in `BaseLayout.astro`, hardcoded to the site owner — not derived from translations.
- **No JavaScript router:** Language switching is a plain `<a href>` navigation, not a client-side route change. This keeps the site fully functional without JS.
- **GitHub Pages deployment** with base path `/portfolio` configured in `astro.config.mjs`. All internal URL construction normalises the base via `import.meta.env.BASE_URL.replace(/\/?$/, '/')`.
